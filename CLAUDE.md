# 宏观仪表盘 CLAUDE.md

## 项目基本信息

- **项目名称**：宏观仪表盘 (Macro Dashboard)
- **类型**：量化投研系统
- **核心功能**：数据采集→信号计算→观点映射→自动简报
- **技术栈**：React + ECharts（前端）、FastAPI（后端）、FRED/AKShare/yfinance（数据）

## 开工流程

1. 读 SPEC.md 理解完整需求
2. 读 docs/ 目录下的需求文档（数据字典、观点映射、架构方案）
3. 确认数据字典字段名规范：后端用英文 ID，对客展示用中文简称

## 字段命名规范

**后端存储/传输**：数据字典英文 ID（如 `cn_pmi_official`、`cn_us_10y_spread`）
**对客展示**：中文简称（如"官方制造业PMI"、"中美利差"）

## 信号合成规则

详见 SPEC.md Section 5：
- 5个因子：中美利差、M1-M2、VIX、北向资金、信用利差
- 每因子打分 -2 到 +2
- 权重：利差20%、M1-M2 25%、VIX 15%、北向15%、信用25%
- 仓位映射：总分 → 权益仓位%

## 重要约束

1. **本地文件必须及时 commit**：`git reset --hard` 会丢弃未提交文件，每次修改重要文件后立即 commit
2. **不做模拟数据 fallback**：数据不可用时显示空态，不静默 fallback
3. **数据新鲜度标注**：月频数据需标注数据月份
4. **观点映射必须有**：每个指标卡片需要显示 signal/view/narrative，不是只显示数字

## 沟通原则

- 简洁，每次回复不超过 300 字
- 先给结论，再给解释
- 不确定的问题先列选项，不自己瞎猜
