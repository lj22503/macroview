# 中国视角全球大类资产宏观量化仪表盘 —— 架构设计方案


## 一、设计目标与核心原则

### 1.1 设计目标

| 目标 | 说明 |
|---|---|
| **全链路自动化** | 从数据采集 → 指标计算 → 信号生成 → 观点输出，全流程无需人工干预 |
| **中国视角优先** | 所有指标围绕中国投资者的资产配置决策链组织，中美利差、USD/CNH、北向资金为核心锚点 |
| **可解释性** | 每个指标输出必须附带“观点映射”（看多/看空/中性），直接服务客户沟通 |
| **零商业数据依赖** | 100%基于公开数据源（FRED + AKShare + yfinance），无需Bloomberg/Reuters等付费终端 |
| **Agent原生** | 架构设计充分考虑LLM Agent的接入，支持自然语言查询与自动报告生成 |

### 1.2 设计原则

参考行业实践，宏观量化系统的核心在于**将数据采集、信号计算与决策输出解耦**。本方案采用**四层架构**：**数据采集层 → 计算与信号层 → 展示层 → Agent交互层**。同时借鉴“因果数据湖”的分层设计理念——原始数据入湖后经清洗、计算形成可直接使用的信号层，以及“混合架构”思路——确定性计算（规则引擎）与模型辅助（AI增强）并存。

## 三、各层详细设计

### 3.1 数据采集层

#### 3.1.1 三大数据源适配器

| 适配器 | 覆盖指标 | 技术实现 | 更新策略 |
|---|---|---|---|
| **FRED 适配器** | VIX、MOVE、DXY、美债收益率、核心PCE、ISM PMI、信用利差、比特币 | `requests` + FRED REST API（需注册免费Key） | 每日08:30全量拉取；盘中按需 |
| **AKShare 适配器** | 中国PMI、CPI/PPI、社融、M1/M2、LPR、北向/南向资金、中债收益率 | `akshare` Python库，零配置 | 每日09:00拉取；月频数据发布日重点刷新 |
| **yfinance 适配器** | 全球股指实时价格、黄金、原油、外汇对（USD/CNH、USD/JPY） | `yfinance` Python库，零配置 | 盘中每5分钟轮询 |

> **数据新鲜度设计**：参考行业实践，FRED等日频数据按小时缓存（cached hourly），避免API配额耗尽。AKShare作为“分布式爬虫集合”，请求直接路由到源站（东财、新浪等），不适合高频轮询，建议按需调用。

#### 3.1.2 事件流适配器（模块七）

| 数据源 | 覆盖内容 | 技术实现 |
|---|---|---|
| **Trading Economics API** | 全球经济日历（发布日、预期值、前值） | REST API，免费版500次/月 |
| **金十数据/财联社** | 实时快讯（政策、地缘、数据） | RSS订阅 + 关键词过滤爬虫 |
| **央行官网轮询** | 中国人民银行、美联储公告 | 定时轮询新闻稿页面 |

### 3.2 计算与信号层（核心）

#### 3.2.1 指标计算引擎

**功能**：将原始数据转化为标准化指标字段。

**输入**：三大适配器拉取的原始数据（如 `DGS10`、`DGS2`）
**计算逻辑**：
- 中美利差 = `cn_10y_yield - us_10y_yield`
- M1-M2剪刀差 = `M2_增速 - M1_增速`
- 北向资金3日累计 = `sum(近3日每日净流入)`
- 衍生指标：各指标的 **Z-score**（过去252个交易日标准化）、**MA20**（20日均线）、**环比变化**

**输出**：标准化指标表（见数据字典，含字段名、数值、时间戳）

#### 3.2.2 信号合成引擎

**功能**：将多个指标加权合成顶层信号（如“RISK ON/OFF”）。

**合成逻辑**（示例）：

```
RISK_ON_SCORE = 
    w1 * 中美利差信号 + 
    w2 * M1-M2剪刀差信号 + 
    w3 * VIX信号 + 
    w4 * 北向资金信号 + 
    w5 * 信用利差信号

其中每个“信号”为 -2 到 +2 的离散值（来自观点映射表）
```

**输出**：
- `overall_bias`：RISK ON / RISK OFF / NEUTRAL
- `confidence`：置信度（0-100%）
- `primary_driver`：主要驱动因素（如“利率 + 美元 + 北向资金”）

#### 3.2.3 观点映射引擎

**功能**：将数值自动匹配到观点区间，生成客户话术。

**实现方式**：规则引擎（Rule Engine），每个指标配置阈值区间及对应话术模板。

**示例**（中美利差）：
```python
{
    "indicator": "cn_us_10y_spread",
    "rules": [
        {"condition": "> -150 and trend == '收窄'", 
         "view": "看多", 
         "narrative": "资本回流中国，人民币资产吸引力上升"},
        {"condition": "< -250 and trend == '走阔'", 
         "view": "看空", 
         "narrative": "资本外逃警报，规避外资重仓股"}
    ]
}
```

**输出**：每个指标的 `signal`（+2/-2）、`view`（看多/看空/中性）、`narrative`（客户话术）

#### 3.2.4 缓存与快照服务

- **Redis缓存**：存储计算后的指标值，TTL按指标类型设置（日频指标24小时，实时指标5分钟）
- **快照服务**：每次全量计算后生成“宏观快照”（Snapshot），包含所有指标的当前值+信号+观点，便于历史回溯和A/B测试

### 3.3 Agent 交互层

#### 3.3.1 架构定位

参考“三引擎协同”的量化系统架构，Agent层作为**智能编排器**，负责理解用户意图、路由到对应数据管道、合成最终回答。同时借鉴“代理接口 × 工具路由器 × 工作流编排器”的设计模式，实现自然语言到结构化查询的自动转化。

#### 3.3.2 核心能力

| 能力 | 说明 | 技术实现 |
|---|---|---|
| **自然语言查询** | 用户问“现在中美利差多少？什么信号？”→ Agent自动查询并回答 | LLM + Function Calling |
| **自动报告生成** | 每日/每周自动生成《宏观观点简报》 | 模板引擎 + 观点映射表填充 |
| **异常预警** | 指标突破关键阈值时主动推送 | 规则引擎 + 推送服务 |
| **多轮对话** | 支持追问（如“为什么是看多？”“主要驱动因素是什么？”） | 对话上下文管理 |

#### 3.3.3 技术选型建议

| 组件 | 推荐方案 | 说明 |
|---|---|---|
| **LLM** | DeepSeek / Qwen / Gemini | 中文理解能力强，成本可控 |
| **编排框架** | LangChain / Dify / 自建 | 管理工具调用与对话流程 |
| **Function Calling** | 将每个指标定义为独立Tool | Agent可自主选择调用哪些指标 |
| **知识库** | 向量数据库（Chroma/Qdrant） | 存储历史观点与研究报告，支持RAG |

| **模块三：全球宏观** | 美国ISM PMI、核心PCE、美联储资产负债表 | 日频 |
| **模块四：美元与流动性** | DXY、中美利差、USD/CNH、USD/JPY | 实时 |
| **模块五：全球核心资产** | 标普500、沪深300、黄金、原油、10Y美债 | 实时 |

## 四、数据流时序设计

### 4.1 每日调度（建议）

| 时间 | 任务 | 说明 |
|---|---|---|
| **08:30** | 全量数据拉取 | FRED日频 + AKShare国内宏观（刷新昨日收盘数据） |
| **08:45** | 指标计算 | 计算所有衍生指标（利差、剪刀差、MA等） |
| **09:00** | 信号合成 | 运行规则引擎，生成RISK ON/OFF及所有观点 |
| **09:15** | 快照保存 | 保存当前宏观快照，供Agent查询 |
| **09:30** | 简报生成 | 自动生成《今日宏观观点》简报（推送至投顾） |
| **盘中（每5分钟）** | 价格轮询 | 仅刷新yfinance实时价格（股指、汇率、商品） |
| **盘中（事件触发）** | 新闻监测 | 关键词匹配到高风险事件时，自动推送预警 |

### 4.2 特殊日期处理

| 场景 | 处理方式 |
|---|---|
| **中国PMI发布日（月末09:30）** | Agent提前标记为“高风险日”，发布后立即刷新并重算信号 |
| **美国CPI/NFP发布日** | 同上，关注北京时间20:30窗口 |
| **FOMC决议日** | 次日08:30全量刷新，重点比对点阵图变化 |


## 五、Agent集成方案（核心亮点）

### 5.1 设计理念

Agent不只是“问答机器人”，而是**宏观研究的自动化协作者**。它应该能：
1. **主动拉取**：按调度自动执行数据采集与信号计算
2. **被动响应**：回答用户关于任何指标/信号/观点的自然语言问题
3. **主动预警**：指标突破阈值时推送警报

### 5.2 Function Calling 定义（示例）

将每个核心指标定义为Agent可调用的Tool：

```python
{
    "name": "get_cn_us_spread",
    "description": "获取当前中美10年期国债利差（中国-美国），以及对应的市场观点",
    "parameters": {
        "period": {"type": "string", "enum": ["current", "historical"]}
    },
    "response": {
        "value": -2.05,  # 单位：%
        "signal": "看多",
        "narrative": "利差收窄趋势确立，资本回流中国，人民币资产吸引力上升",
        "trend": "收窄",
        "updated_at": "2026-06-18 08:45:00"
    }
}
```

### 5.3 自动简报生成模板

**《今日宏观观点》**（每日09:30自动生成）

```
【今日宏观概览】
风险偏好：RISK ON（置信度 71%）
主要驱动：中美利差收窄 + 北向资金连续流入 + VIX低位稳定

【中国内核】
- PMI：51.2（扩张加速）→ 经济动能增强，超配权益
- M1-M2剪刀差：-4.8%（收窄）→ 资金活化，牛市前兆信号

【外部环境】
- 美国核心PCE：2.4%（<2.5%）→ 降息窗口打开，全球Risk On
- 中美利差：-205bp（较上周收窄15bp）→ 资本回流趋势确认

【配置建议】
- A股：超配（沪深300目标仓位 70%）
- 债券：低配（利率上行风险）
- 黄金：标配（实际利率下行支撑）
- 现金：低配（机会成本高）

【风险提示】
- 高收益债利差：420bp（正常区间，暂无信用风险）
- VIX：16.2（正常）
```


## 六、部署与运维方案

### 6.1 部署架构

| 组件 | 部署方式 | 说明 |
|---|---|---|
| **数据采集** | Python脚本 + Cron定时任务 | 轻量级，单机可运行 |
| **计算引擎** | FastAPI后端服务 | 提供REST API供前端和Agent调用 |
| **缓存** | Redis | 存储计算后的指标快照 |
| **数据库** | SQLite / DuckDB（本地）或 TimescaleDB（云端） | 存储历史数据，支持回测 |
| **前端** | 静态部署（Vercel/Netlify/Nginx） | 纯前端，通过API获取数据 |
| **Agent** | LangChain / Dify 编排 | 独立服务，调用后端API |

### 6.2 成本估算

| 项目 | 成本 | 说明 |
|---|---|---|
| FRED API | 免费 | 需注册Key，无调用次数限制 |
| AKShare | 免费 | 开源库，零配置 |
| yfinance | 免费 | 开源库，零配置 |
| 服务器 | 约 ¥100-300/月 | 单台云服务器（2C4G）足够 |
| Redis | 免费（本地） | 或使用云Redis（约 ¥50/月） |
| LLM API | 约 ¥0.5-2/天 | 按调用量计费（DeepSeek/Qwen） |
| **合计** | **约 ¥150-500/月** | 远低于Bloomberg终端（$2,000+/月） |

### 6.3 监控与告警

- **数据源健康检查**：每日首次拉取后验证数据新鲜度（如有指标为空或超过预期时效，触发告警）
- **API配额监控**：FRED API调用次数接近上限时预警
- **信号一致性校验**：若RISK ON/OFF信号与多数子指标信号矛盾，触发人工复核

## 八、架构优势总结

| 维度 | 本方案优势 |
|---|---|
| **成本** | 完全基于免费公开数据源，月度运营成本 < ¥500 |
| **可维护性** | 四层解耦，每层可独立升级（如替换数据源无需重写前端） |
| **可解释性** | 每个指标输出自带观点映射，客户沟通零障碍 |
| **Agent原生** | 从设计之初即为LLM集成预留接口，支持自然语言交互 |
| **扩展性** | 新增指标只需在数据采集层添加适配器 + 计算层添加规则 |
| **中国视角** | 所有指标围绕中国投资者的决策链组织，非简单照搬海外框架 |
