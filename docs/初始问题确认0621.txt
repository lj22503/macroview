## 一、后端 API 端点设计方案

建议采用 **“统一全景接口 + 独立模块接口”** 的双层设计，兼顾首屏加载效率和模块独立查询需求。

### 1.1 统一全景接口（推荐首屏使用）

`GET /api/v1/dashboard`

返回完整的仪表盘数据，一次性满足前端所有模块渲染需求。

```json
{
  "meta": {
    "updated_at": "2026-06-21T09:00:00+08:00",
    "data_date": "2026-06-20",
    "status": "success"
  },
  "overview": {
    "bias": "RISK_ON",
    "confidence": 68,
    "score": 1.2,
    "primary_driver": ["中美利差收窄", "M1-M2剪刀差改善", "VIX正常"],
    "suggested_equity": 65,
    "narrative": "国内流动性改善叠加外部压力缓解，温和复苏延续..."
  },
  "china_core": { ... },
  "global_macro": { ... },
  "fx_liquidity": { ... },
  "assets": { ... },
  "risk_monitor": { ... },
  "events": [ ... ]
}
```

**优点**：一次请求，全量渲染；缺点：数据量大（但60+指标JSON通常<50KB，可接受）。

### 1.2 独立模块接口（按需使用）

| 端点 | 返回内容 | 适用场景 |
|---|---|---|
| `GET /api/v1/overview` | 顶层决策锚点（bias/confidence/score/primary_driver/仓位建议/叙事文本） | 顶部状态栏单独刷新 |
| `GET /api/v1/china-core` | PMI、CPI/PPI、社融、M1-M2剪刀差、LPR | 国内宏观模块 |
| `GET /api/v1/global-macro` | 美国ISM PMI、核心PCE、美联储资产负债表、ON RRP | 外部环境模块 |
| `GET /api/v1/fx-liquidity` | DXY、中美利差、USD/CNH、USD/JPY | 汇率与流动性模块 |
| `GET /api/v1/assets` | 标普500、沪深300、黄金、原油、10Y美债 | 核心资产行情模块 |
| `GET /api/v1/risk` | VIX、MOVE、高收益债利差、A股隐含波动率 | 风控监测模块 |
| `GET /api/v1/events` | 经济日历、实时快讯 | 事件流模块 |

## 二、信号合成权重

RISK_ON_SCORE 的计算分为两步：

**Step 1：各因子独立打分（-2 到 +2）**

| 因子 | 权重符号 | 打分规则 | 当前状态（2026-06-21） | 得分 |
|---|---|---|---|---|
| 中美利差信号 | w1 | +2（利差>-150bp且收窄）→ +1（-200~-150bp且收窄）→ 0（-250~-200bp持平）→ -1（<-250bp走阔）→ -2（<-300bp加速走阔） | 约-205bp，近月收窄 | **+1** |
| M1-M2剪刀差信号 | w2 | +2（收窄且>-3%）→ +1（收窄但<-3%）→ 0（持平）→ -1（走阔但>-8%）→ -2（走阔且<-8%） | 持续收窄，约-4.8% | **+1** |
| VIX信号 | w3 | +2（<14）→ +1（14-18）→ 0（18-22）→ -1（22-28）→ -2（>28） | 约16.5 | **+1** |
| 北向资金信号 | w4 | +2（3日累计>100亿）→ +1（>50亿）→ 0（-50~+50亿）→ -1（<-50亿）→ -2（<-100亿） | 近期温和流入 | **+1** |
| 信用利差信号 | w5 | +2（<350bp）→ +1（350-450bp）→ 0（450-550bp）→ -1（550-700bp）→ -2（>700bp） | 约420bp | **+1** |

**Step 2：加权合成**

各因子权重基于当前宏观背景（中美利差倒挂+国内弱复苏）设定：

| 因子 | 权重 | 理由 |
|---|---|---|
| 中美利差信号 | **20%** | 决定外资行为，是当前市场核心矛盾（倒挂修复节奏） |
| M1-M2剪刀差信号 | **25%** | 决定国内增量资金，权重最高——它是“水牛”能否启动的根本 |
| VIX信号 | **15%** | 反映全球风险偏好，是仓位调整的辅助指标 |
| 北向资金信号 | **15%** | 反映外资即时态度，但对冲属性强（与利差高度相关），不宜过高权重 |
| 信用利差信号 | **25%** | 系统性风险开关，权重大但平时为0分，关键时刻发挥截断作用 |

**Step 3：代入计算**

```
RISK_ON_SCORE =
    0.20 × (+1) +
    0.25 × (+1) +
    0.15 × (+1) +
    0.15 × (+1) +
    0.25 × (+1)
= 0.20 + 0.25 + 0.15 + 0.15 + 0.25
= 1.0
```

> 注：上述得分基于2026年6月21日宏观状态估算。实际运行中，VIX和北向资金需替换为实时或最新日频数据。

若某因子信号为0分，则其贡献为0。信用利差当前420bp处于“正常”区间，得+1分而非0分，因为低于450bp属于健康水平。若利差升至550bp以上，该因子分数将由正转负，对总分产生拖累。

### 仓位映射逻辑

| 总分范围 | 权益仓位 | 叙事标签 |
|---|---|---|
| +1.5 ~ +2.0 | 80% ~ 100% | 全面扩张 |
| +0.5 ~ +1.5 | 60% ~ 80% | 温和复苏 |
| -0.5 ~ +0.5 | 40% ~ 60% | 结构分化 |
| -1.5 ~ -0.5 | 20% ~ 40% | 收缩压力 |
| -2.0 ~ -1.5 | 0% ~ 20% | 系统性风险 |

### 五、Agent 生成简报的伪代码

```python
def generate_daily_brief(score_result, event_data):
    """
    score_result: 来自信号合成引擎的输出
    event_data: 来自事件流模块的近期待发布数据
    """
    # 1. 映射宏观标签
    label = map_score_to_label(score_result.total)
    
    # 2. 生成各因子解读
    factor_comments = []
    for factor in score_result.factors:
        comment = {
            "name": factor.name,
            "signal": factor.signal,
            "trend": factor.trend,
            "comment": factor_comment_template(factor)
        }
        factor_comments.append(comment)
    
    # 3. 提取主要驱动（正分因子中权重最高的2-3个）
    primary_drivers = extract_drivers(factor_comments)
    
    # 4. 生成配置建议
    allocation = map_score_to_allocation(score_result.total)
    
    # 5. 风险提示（从事件流中提取近期待发布的高风险事件）
    risk_warning = extract_risk_warning(event_data)
    
    # 6. 一句话核心观点（LLM生成或模板拼接）
    one_sentence = generate_summary(factor_comments, primary_drivers)
    
    # 7. 渲染模板
    brief = render_template({
        "date": today(),
        "label": label,
        "score": score_result.total,
        "equity_pct": allocation.equity,
        "one_sentence": one_sentence,
        "factors": factor_comments,
        "primary_drivers": primary_drivers,
        "bond_pct": allocation.bond,
        "gold_pct": allocation.gold,
        "cash_pct": allocation.cash,
        "risk_warning": risk_warning
    })
    
    return brief
```

### 六、⚠️ 数据新鲜度约束

以下两个因子受数据发布频率限制，简报生成时需遵守：

| 因子 | 发布频率 | 延迟说明 | 简报中的处理方式 |
|---|---|---|---|
| **M1-M2剪刀差** | 月频（次月中旬） | 最新值为上月数据 | 在简报中标注“M1-M2（{{数据月份}}）”，当月无新数据时沿用上月值 |
| **信用利差** | 日频 | FRED延迟约1-2个交易日 | 在简报底部标注“信用利差数据截至{{数据日期}}” |

具体到当前（2026年6月21日）：
- **M1-M2**：使用5月数据（约-4.8%），简报中标注`M1-M2剪刀差（5月）-4.8%`
- **信用利差**：若FRED最新数据仅到6月19日，简报中标注`高收益债利差420bp（数据截至6月19日）`

简报生成时需对每个指标标注`data_date`，避免用户误以为所有数据均为“今天”的实时数据。
